Template_Method : 
알고리즘의 구조를 메소드에 정의하고
하위 클래스에서 알고리즘 '구조의 변경없이 알고리즘을 재정의' 하는 패턴

알고리즘을 여러 단계로 쪼갠다.
각 단계를 메소드를 선언한다.
템플릿 메소드를 만들고 위의 단계를 호출해서 처리한다.

예시)
게임의 '접속(login)'을 구현해주세요.
각 단계 : 보안 - 인증 - 권한 - 접속
(각 단계에 대한 정의는 노출이 되지 않지만 하위 클래스에서 정의해야 함 = protected)

추상클래스 내부의 순서와 처리 정리는 특정 메소드(템플릿 메소드)가 규정을 해논다.
따라서 실제 구현 클래스(상속받는 자식 클래스)는 구현부를 완성하고,
템플릿 메소드로 실행하면 된다.
